# \# MARL-LEO-Routing

# 

# > \*\*研究方向\*\*：在高动态的低轨卫星网络（LEO Satellite Networks）中实现快速、分布式、协同的动态路由。

# 

# \## 1. 核心问题

# 

# 1\.  \*\*局部信息下的快速路由决策\*\*

# &#x20;   -   卫星数量大、拓扑变化快，全局链路状态更新开销高、收敛慢。

# &#x20;   -   \*\*目标\*\*：设计仅依赖局部观测的分布式路由机制，每颗卫星能根据邻居状态自主选择下一跳。

# 

# 2\.  \*\*多智能体之间的有效协作\*\*

# &#x20;   -   各卫星独立追求局部最优，易导致全局拥塞、路径震荡或流量集中。

# &#x20;   -   \*\*目标\*\*：通过协同奖励和全局优化，实现“局部决策 + 邻居交互 + 全局目标一致”的协作路由。

# 

# 3\.  \*\*队列积压与拥塞感知\*\*

# &#x20;   -   传统路由只看传播时延/跳数，但卫星网络中真正导致时延上升的是节点队列积压和链路拥塞。

# &#x20;   -   \*\*目标\*\*：在路由决策中显式引入队列状态、负载和拥塞惩罚，实现队列感知的动态路由。

# 

# \## 2. 核心技术方案：集中训练，分布式执行 (CTDE)

# 

# 采用多智能体深度强化学习（MARL）中的 \*\*CTDE 框架\*\*：

# 

# \-   \*\*训练阶段（地面）\*\*：使用全局网络状态和\*\*全局协同奖励函数\*\* $R\_{global}$，集中训练全局策略。

# \-   \*\*执行阶段（星上）\*\*：每颗卫星复制训练好的轻量策略模型，仅依据\*\*局部观测\*\*和\*\*邻居信息\*\*独立做出下一跳决策。

# \-   \*\*在线自适应\*\*：部署后，卫星可依据本地观测和\*\*本地奖励函数\*\* $r\_i$ 持续微调策略。

# 

# \### 2.1 系统模型

# 

# \-   \*\*时变图模型\*\*：将动态网络抽象为 $G(t) = (V, E(t), X(t))$

# &#x20;   -   $V = \\{1, 2, \\dots, N\\}$: 卫星节点集合。

# &#x20;   -   $E(t) = \\{(i,j) \\mid \\text{link}\_{i \\to j} \\text{ is available at } t\\}$: 时变星间链路。

# &#x20;   -   $X(t) = \\{x\_i(t), e\_{ij}(t)\\}$: 全网节点与链路状态信息。

# \-   \*\*智能体定义\*\*：每颗卫星即为一个智能体 $Agent\_i \\in V$，共 $N$ 个。

# 

# \### 2.2 决策流程（执行阶段）

# 

# 1\.  \*\*局部观测\*\*：卫星 $i$ 获取一跳邻居状态。

# &#x20;   $$o\_i(t) = \\{x\_i(t), x\_j(t), e\_{ij}(t) \\mid j \\in N\_i(t)\\}$$

# &#x20;   > \*\*待定\*\*：观测范围是否扩展为两跳邻居 $N\_i^{2-hop}(t)$？这会增加决策质量但增加通信开销。

# 

# 2\.  \*\*状态空间\*\*：智能体决策所用的特征输入。

# &#x20;   $$s\_i(t) = \\{q\_i(t), b\_{ij}(t), d\_{ij}(t), l\_{ij}(t), r\_{ij}(t), \\tau\_{ij}(t) \\mid j \\in N\_i(t)\\}$$

# &#x20;   -   $q\_i(t)$: 队列长度

# &#x20;   -   $b\_{ij}(t)$: 链路剩余带宽 ($b\_{ij} = R\_{ij} - f\_{ij}$)

# &#x20;   -   $d\_{ij}(t)$: 传播时延 ($d\_{ij} = |ij|/c$)

# &#x20;   -   $l\_{ij}(t)$: 链路负载率 ($l\_{ij} = f\_{ij}/R\_{ij}$)

# &#x20;   -   $r\_{ij}(t)$: 链路可靠性 ($r\_{ij} = 1 - P\_{out}^{ij}$)

# &#x20;   -   $\\tau\_{ij}(t)$: \*\*链路剩余可用时间\*\*。用于避免向即将断开的链路发送数据。

# 

# 3\.  \*\*邻居信息交换\*\*：通过轻量级 Hello 包交换状态。

# &#x20;   $$m\_i(t) = \\{q\_i(t), l\_i(t), a\_i(t)\\}$$

# 

# 4\.  \*\*动作选择（带掩码）\*\*：从有效邻居中依据\*\*链路代价\*\* $C\_{ij}(t)$ 选择下一跳。

# &#x20;   $$a\_i(t) \\in N\_i^{valid}(t) = \\{j \\in N\_i(t) \\mid \\tau\_{ij}(t) > \\tau\_{min}, r\_{ij}(t) > r\_{min}, b\_{ij}(t) > b\_{min}\\}$$

# &#x20;   $$a\_i(t) = \\arg\\min\_{j \\in N\_i^{valid}(t)} C\_{ij}(t)$$

# &#x20;   -   \*\*链路代价函数\*\*：综合衡量备选路径的优劣。

# &#x20;       $$C\_{ij}(t) = \\alpha \\cdot d\_{ij}(t) + \\beta \\cdot q\_j(t) + \\gamma \\cdot l\_{ij}(t) + \\delta \\cdot \\frac{1}{\\tau\_{ij}(t) + \\epsilon}$$

# 

# 5\.  \*\*执行策略\*\*：所有卫星共享同一个策略网络 $\\pi\_\\theta$。

# &#x20;   $$\\pi\_i = \\pi\_\\theta, \\quad \\forall i \\in V$$

# &#x20;   $$a\_i(t) = \\pi\_\\theta(o\_i(t))$$

# 

# \### 2.3 奖励函数设计（训练阶段）

# 

# 1\.  \*\*全局协同奖励函数\*\*：训练时对齐所有智能体的目标，避免陷入局部最优。

# &#x20;   $$R\_{global}(t) = -\\alpha D(t) - \\beta Q(t) - \\gamma L(t) - \\delta H(t) + \\eta T(t) - \\mu O(t)$$

# &#x20;   -   $D(t)$: 平均端到端时延

# &#x20;   -   $Q(t)$: 全网队列积压

# &#x20;   -   $L(t)$: 负载不均衡度

# &#x20;   -   $H(t)$: 路由切换次数

# &#x20;   -   $T(t)$: 吞吐量 (正向奖励)

# &#x20;   -   $O(t)$: 控制消息开销

# 

# 2\.  \*\*本地即时奖励函数\*\*：用于星上在线微调，引导数据包高效、可靠地前进。

# &#x20;   $$r\_i(t) = -d\_{ij}(t) - \\beta q\_j(t) - \\gamma l\_{ij}(t) + \\eta \\cdot \\mathbb{1}\[j = d] - \\delta \\cdot \\mathbb{1}\[j \\in \\mathcal{P}\_p]$$

# &#x20;   > \*\*待定\*\*：是否需要加入基于排队论的“队列增长趋势”惩罚项 $-\\beta\_2 \\max(0, \\Delta q\_j(t))$，以预测式地规避拥塞？

# 

# \### 2.4 关键文献支撑

# 

# 1\.  \*\*分布式架构与低复杂度\*\*：QRLSN (CJA 2022) 证明了分布式 Q-learning 在卫星网络中的有效性，提供 $O(n)$ 路由维护复杂度。

# 2\.  \*\*队列感知与拥塞控制\*\*：

# &#x20;   -   arXiv 2023 (Soret et al.) 指出 LEO 时延在拥塞时由队列主导，并设计了队列感知的指数惩罚奖励函数。

# &#x20;   -   SDRP-DQL (2026) 提出基于排队论的拥塞趋势预测奖励。

# 3\.  \*\*CTDE 与协作\*\*：arXiv 2026 (Liaq et al.) 提出了队列感知的 MA-DRL 框架，采用全局 Q 网络训练与分布式执行。

# 

# \## 3. 评价指标

# 

# \-   \*\*时延与效率\*\*：平均端到端时延、吞吐量、丢包率、投递成功率

# \-   \*\*拥塞与负载\*\*：平均队列长度、负载均衡度

# \-   \*\*稳定性与开销\*\*：路由切换次数、故障恢复时间、路由开销、收敛时间

# \-   \*\*可行性\*\*：计算复杂度
